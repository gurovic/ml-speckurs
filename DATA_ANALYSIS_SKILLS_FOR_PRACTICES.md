# Навыки анализа данных, которые реально используются в ответах практик ML

Документ составлен **по готовым ответам в практических ноутбуках**, а не по формулировкам заданий. В список попали навыки из предыдущего модуля по анализу данных, которые фактически используются в code-ячейках и markdown-ответах практик чётных уроков. Навыки отсортированы по логике изучения: от базовой работы с таблицами и массивами к визуализации и интерпретации результатов.

Источник: обычные практические ноутбуки уроков 22, 24, 26, 28, 30, 32, 34 и 36. Файлы с `hand` в названии не использовались и не изменялись.

## Общая лестница навыков

### 1. Базовая работа с данными

- Понимать разницу между таблицей `DataFrame`, столбцом `Series`, массивом `numpy.ndarray` и списком Python.
- Читать данные из файлов: `pd.read_csv`, `json.loads`, `Path.read_text`.
- Смотреть первые строки и структуру таблицы: `head`, `info`, `shape`, `describe`.
- Отличать признаки от целевой переменной и исключать служебные столбцы вроде `id`.
- Понимать, что одна строка таблицы соответствует одному объекту наблюдения.

### 2. Выбор и создание столбцов

- Выбирать столбцы по именам и спискам имён.
- Создавать новые признаки через арифметику столбцов.
- Работать с копиями таблиц через `copy`, чтобы не портить исходные данные случайными изменениями.
- Добавлять индикаторы 0/1 через логические условия и `astype(int)`.
- Использовать `np.log1p`, деление, суммы по строкам и базовые математические преобразования.

### 3. Типы данных и пропуски

- Разделять числовые и категориальные признаки через `select_dtypes`.
- Находить пропуски через `isna`, проверять заполненность через `notna`.
- Заполнять пропуски статистикой, посчитанной на train: `median`, `fillna`.
- Понимать, почему статистики для обработки нельзя считать по всей таблице до split.

### 4. Группировки и агрегаты

- Считать распределения категорий через `value_counts`.
- Считать доли через `value_counts(normalize=True)` и средние значения через `mean`.
- Использовать `groupby` для сравнения групп.
- Собирать небольшие таблицы результатов из словарей и списков: `pd.DataFrame`, `pd.Series(...).to_frame`.
- Сортировать таблицы результатов и выбирать лучшую строку через `sort_values`, `idxmin`, `idxmax`, `loc`, `iloc`.

### 5. Работа с датами и категориями

- Извлекать компоненты даты через `.dt`, например месяц.
- Считать разницу дат и переводить её в дни.
- Кодировать циклические признаки через `sin` и `cos`.
- Понимать one-hot кодирование категорий как превращение значения в несколько бинарных столбцов.
- Понимать порядковое кодирование категорий, когда у значений есть естественный порядок.

### 6. NumPy для массивов и экспериментов

- Создавать массивы и диапазоны: `np.array`, `np.linspace`, `range`.
- Работать с формой массива: `reshape`, `flatten`.
- Считать суммы, средние, стандартные отклонения и уникальные значения: `sum`, `mean`, `std`, `unique`.
- Использовать генератор случайных чисел `np.random.default_rng`.
- Создавать выборки с возвращением через `rng.choice`.
- Использовать `np.where` для векторного выбора значений.
- Понимать оси вычислений: `axis=0` и `axis=1`.

### 7. Визуализация

- Строить базовые графики `matplotlib`: `bar`, `plot`, `scatter`, `boxplot`, `imshow`.
- Настраивать размер графика через `plt.figure(figsize=...)`.
- Подписывать графики: `title`, `xlabel`, `ylabel`, `legend`, `xticks`, `yticks`.
- Ограничивать шкалу и делать график читаемым: `ylim`, `grid`, `rotation`, `ha`.
- Использовать `plt.show`, чтобы явно завершать график.
- Читать графики как инструмент сравнения моделей, ошибок и групп данных.

### 8. Интерпретация результатов

- Сравнивать значения метрик по таблице или графику.
- Формулировать текстовый вывод на основе чисел, а не общих рассуждений.
- Отличать train, validation и test и понимать, почему сравнения делаются на validation.
- Видеть признаки утечки данных: когда в признаки попадает цель или информация из будущего.
- Объяснять, почему конкретный признак, график или метрика полезны для задачи.
